# Module 6 — 6StormStatistics

## Technical Module Documentation

---

# Purpose

`6StormStatistics.py` extracts descriptive statistics from the geomagnetic storm catalogue generated by Module 4.

The module calculates storm duration, recovery duration, seasonal occurrence, and Solar Cycle classification, providing temporal characteristics of geomagnetic storms throughout the study period.

Unlike previous modules that focus on data preprocessing or statistical correlation, this module summarizes storm behaviour in terms of time and occurrence patterns.

Dst_Storm_Events.csv

# Output

The module generates a structured collection of storm statistics including:

* Storm duration
* Recovery duration
* Season of occurrence
* Solar cycle assignment

These statistics may be printed to the console or exported for later reporting and visualization.

---

# Scientific Operations

The module performs the following sequence:

1. Load storm-event catalogue.
2. Read storm start times.
3. Read storm peak times.
4. Read storm recovery times.
5. Calculate total storm duration.
6. Calculate recovery duration.
7. Assign meteorological season.
8. Classify Solar Cycle.
9. Store storm metadata.
10. Display statistical summaries.

---

# Storm Duration

Storm duration is calculated as

```text
Duration = Recovery Time − Start Time
```

and is expressed in hours.

---

# Recovery Duration

Recovery duration is calculated as

```text
Recovery = Recovery Time − Peak Time
```

This quantity estimates the time required for the geomagnetic field to recover following maximum storm intensity.

---

# Seasonal Classification

Each storm is assigned to one of four seasons according to its occurrence date.

| Season        | Date Range                 |
| ------------- | -------------------------- |
| Spring        | 21 March – 20 June         |
| Summer        | 21 June – 22 September     |
| Autumn (Fall) | 23 September – 20 December |
| Winter        | 21 December – 20 March     |

This classification enables investigation of seasonal variability in geomagnetic activity.

---

# Solar Cycle Classification

The module includes automatic Solar Cycle identification based on observation year.

| Year Range | Solar Cycle    |
| ---------- | -------------- |
| 1996–2008  | Solar Cycle 23 |
| 2009–2019  | Solar Cycle 24 |
| 2020–2030  | Solar Cycle 25 |

Within this project, all analysed storms belong to **Solar Cycle 24**.

# Error Handling

The module automatically skips:

* malformed timestamps,
* incomplete storm records,
* invalid recovery times,
* missing event information.

Processing continues using all valid storm events.

This module transforms individual storm detections into descriptive statistics, enabling analysis of:

* storm longevity,
* recovery characteristics,
* seasonal variability,
* Solar Cycle occurrence.

These temporal descriptors complement the statistical relationships derived in Module 5 and provide important context for interpreting geomagnetic activity.
