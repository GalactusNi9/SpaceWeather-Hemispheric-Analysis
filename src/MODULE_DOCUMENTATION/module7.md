# Module 7 — 7MergeGroundStationData

## Technical Module Documentation

---

# Purpose

`7MergeGroundStationData.py` combines individually cleaned daily INTERMAGNET datasets into continuous yearly observatory datasets.

After Module 2 has produced cleaned hourly CSV files for each day, this module concatenates them into a single chronological dataset representing an entire calendar year for either the SJG or PIL observatory.

The resulting yearly files provide continuous geomagnetic H-component records required for hemispheric correlation analyses.

---

# Input

The module processes directories containing cleaned daily CSV files generated by Module 2.

Example

```text
PIL/
    pil20140101dmin_cleaned.csv
    pil20140102dmin_cleaned.csv
    ...
    pil20141231dmin_cleaned.csv
```

Each file contains:

* Datetime
* H
* D
* Z
* F

---

# Output

The module generates one continuous yearly dataset.

Example

```text
PIL_2014_Merged_Cleaned.csv
```

containing

```text
Datetime
H
```

The merged dataset consists of hourly H-component observations covering the complete year.

---

# Scientific Operations

The module performs the following sequence:

1. Locate all cleaned daily CSV files.
2. Sort files chronologically.
3. Read each dataset.
4. Detect the H-component column.
5. Extract hourly H observations.
6. Concatenate observations.
7. Verify yearly record length.
8. Pad missing observations when necessary.
9. Trim excess observations if required.
10. Generate continuous hourly timestamps.
11. Export merged yearly dataset.

---

# Expected Dataset Length

For non-leap years, the module expects:

```text
365 × 24 = 8760 hourly observations
```

If fewer observations are available, missing values are padded to preserve temporal continuity.

If additional observations exist, excess records are removed to maintain the expected annual length.

---

# Data Validation

The module automatically:

* detects CSV delimiters,
* identifies the H column,
* ignores malformed rows,
* skips files without H-component measurements.

This improves compatibility with datasets exported from different processing environments.

---

# Timestamp Generation

Rather than relying solely on individual file timestamps, the module reconstructs a continuous hourly timeline beginning at:

```text
01 January
00:00 UTC
```

and extending through the final hour of the selected year.

This guarantees consistent temporal indexing across all yearly datasets.
